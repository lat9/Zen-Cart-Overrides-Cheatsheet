Zen Cart — Overrides Cheatsheet (created by lat9) — v1.1.1

The list below shows the storefront directory structure for Zen Cart v1.3.9 through v1.5.5b. Locations that are used for general overrides (i.e. affecting all templates) are identified by this color and those that are used for template-specific overrides (i.e. affecting only that template) are identified by this color. Questions and/or comments can be directed to the support thread: http://www.zen-cart.com/showthread.php?176529

Change History:

  • v1.0.0, 2011/03/15
    • Initial release.
  • v1.0.1, 2011/03/16
    • /includes/index_filters/YOUR_TEMPLATE is available for template-specific overrides.
    • Added a cautionary notes to the /includes/modules/pages/* and /includes/templates/YOUR_TEMPLATE descriptions.
    • /includes/templates/YOUR_TEMPLATE/product_type/extra_main_template_vars is available for template-specific overrides of variables required for the specific product_type.
  • v1.0.2, 2012/04/30
    • Noted that the directory structure applies for v1.5.0 as well.
  • v1.0.3, 2013-12-04
    • Noted that the directory structure applies for v1.5.1 and v1.5.2 as well.
    • Added regarding file renaming — renaming files to an extension of .bak can raise PCI flags, suggest alternate rename (thanks to dbltoe).
  • v1.1.0, 2014-09-06
    • Zen Cart v1.5.3 introduced auto-loaded observers; see description in /includes/classes/observers.
  • v1.1.1, 2016-11-10
    • Note that the overrides apply to Zen Cart versions up to and including v1.5.5b.
    • Identify the locations where files are loaded in file-system rather than alphabetical order.

Clicking on a link reveals the override opportunities for the associated directory, clicking it again hides the content. If a directory's name is noted*, then the files in that directory are loaded in file-system order … i.e. you cannot predict the order in which the directory's files load.

  • /includes
    • /auto_loaders*
      If you are implementing an observer or other site-wide functionality (such as loading a required class or init_include), create a file named config.yourcustomname.php in this directory to contain your changes. Refer to this Zen Cart thread (opens in a new window) for additional information about overrides.
      • /overrides*
        To change the behavior of a core auto-loader, copy the file from /includes/auto_loaders to this directory and make your changes to affect all pages.
    • /classes
      • /db
      • /observers
        If you are implementing an auto-loaded observer class, create a file named class.yourcustomname.php in this directory to contain your changes. Refer to this Zen Cart thread (opens in a new window) for additional information about overrides.

        Zen Cart v1.5.3 introduced the concept of an auto-loaded observer, a file named auto.yourcustomname.php. Refer to the comments in /includes/init_includes/init_observers.php or this blog post (opens in a new window) for more information.

    • /extra_cart_actions*
      If you want to change the behavior of the shopping_cart actions, create a file named yourcustomname.php in this directory. All .php files in this directory are run prior to running the core shopping_cart actions.
    • /extra_configures*
      All .php files in this directory are unconditionally loaded prior to page, template, and language determination.
    • /extra_datafiles
      Files in this directory are normally used to define additional filenames (if you have defined additional pages for your site) or additional database table names you have defined. All files with a .php extension are unconditionally loaded for each page.
    • /functions
      • /extra_functions
        If you have defined additional functions, create a file named yourfunctionsname.php in this directory to contain your changes. All files with a .php extension are unconditionally loaded for each page.
    • /index_filters
      • /YOUR_TEMPLATE
        To make template-specific changes to any core file in the index_filters directory, copy the file from /includes/index_filters to this directory and make your changes.
    • /init_includes
      If your modification requires a new initialization file (loaded by an auto-loader), create a file named init_yourcustomname.php in this directory. That filename format is conventional; just make sure that the filename is the same as that specified in /includes/auto_loaders/config.yourcustomname.php.
      • /overrides
        To change the behavior of a core initialization file, copy the file from /includes/init_includes to this directory and make your changes to affect all pages.
    • /languages
      • /classic
      • /english
        • /classic
        • /extra_definitions
          • /classic
          • /YOUR_TEMPLATE
            To make template-specific changes to any core file in the extra_definitions directory, copy the file from /includes/languages/english/extra_definitions to this directory and make your changes.
        • /html_includes
          • /classic
          • /YOUR_TEMPLATE
            To make template-specific changes to any core file in the html_includes directory, copy the file from /includes/languages/english/html_includes to this directory and make your changes. Alternatively, you can use your admin’s Define Pages Editor (when you have selected YOUR_TEMPLATE as the active template) to make the changes.
        • /images
        • /modules
          • /order_total
            • /classic
            • /YOUR_TEMPLATE
              To make template-specific changes to any core file in the order_total directory, copy the file from /includes/languages/english/modules/order_total to this directory and make your changes.
          • /payment
            • /classic
            • /YOUR_TEMPLATE
              To make template-specific changes to any core file in the payment directory, copy the file from /includes/languages/english/modules/payment to this directory and make your changes.
          • /shipping
            • /classic
            • /YOUR_TEMPLATE
              To make template-specific changes to any core file in the shipping directory, copy the file from /includes/languages/english/modules/shipping to this directory and make your changes.
        • /YOUR_TEMPLATE
          To make template-specific changes to any core file in the english directory, copy the file from /includes/languages/english to this directory and make your changes.
      • /YOUR_TEMPLATE
        To make template-specific changes to any core file in the base languages directory (e.g. english.php), copy the file from /includes/languages to this directory and make your changes.
    • /local
      If you have a copy of your Zen Cart files on your local machine, create a copy of /includes/configure.php in this directory and make the modifications required to access your local file structure and database. This will allow you to perform local testing using a copy of your hosted implementation. Note: Be sure not to copy this directory to your hosted version!
    • /modules
      • /classic
      • /debug_blocks
      • /order_total
      • /pages
        • *
          For any page, you can add function by creating the file header_phpyourcustomname.php. These files are loaded — after the core header_php.php file — in alphabetical order and are run unconditionally (i.e. for every template selection).

          Notes:
          • If the core header_php.php file processing results in a redirect to a different page, your custom code will never be run.
          • Be careful when renaming an existing header_php.php file for archive purposes; if you rename the file to header_php-backup.php (or similar), that file will still be run (and cause you all kinds of grief). Your best approach is to change the file extension for the backup to something other than .php — like, possibly, your initials or .php~. Don't use .bak, since that might raise PCI-compliance flags.
      • /payment
      • /shipping
      • /sideboxes
        • /classic
        • /YOUR_TEMPLATE
          To make template-specific changes to any core sidebox, copy the file from /includes/modules/sideboxes to this directory and make your changes.
      • /YOUR_TEMPLATE
        To make template-specific changes to any core module, copy the file from /includes/modules to this directory and make your changes.
    • /templates
      • /classic
      • /template_default
      • /YOUR_TEMPLATE
        Here is where the majority of your template-specific files reside; just mimic the directory structure of /includes/templates/template_default for your purposes, including only those directories and files that you are modifying. Start by copying a file from template_default into the same sub-directory under YOUR_TEMPLATE and then make your changes. You should not copy the entire contents of the template_default directory; only copy those files that you are changing for your template — note that you must have a /css sub-directory as all CSS files are template-specific!
        • /product_type/extra_main_template_vars
          You can customize the variables for a specific Zen Cart product_type (e.g. document_general_info, document_product_info, product_free_shipping_info, product_info or product_music_info) by creating one or more .php files in the associated directory. All .php files are run (in alphabetical sort-order) by the product_type page’s main_template_vars.php file.

          For example, all .php files in /includes/templates/YOUR_TEMPLATE/product_info/extra_main_template_vars are run as part of the template-specific processing for the product_info page.
